Question : Discuss the scope of var let and const

In JavaScript var, let, and const are used to declare variables, but they have different scopes and behaviors. Let's discuss the scope of each:

  • var :
  • var declarations are function-scoped.This means that a variable declared with var is only accessible within the function.
  • If a var variable is declared outside of a function it becomes global variable,which means it's accessible throughout the program
  • Variable declared with var are hoisted,that means they are moved to the top their containing function.This can cause an unexpected behavior if you try to access before declare.
  • let :
  • let declaration is block-scoped.That means a variable declared with let only accessable within the block where it is define.
  • let variables are not hoisted.This means you can't access them before their declaration in the code
  • const :
  • Like let, const declarations are block-scoped.
  • The key difference is that once you assign a value to a const variable, you cannot reassign it to a different value. The variable remains constant.
  • However, the contents of an object or an array declared with const can still be modified. The immutability applies only to the reference, not the object's properties or array elements.

    • Question : Tell us the use cases of null and undefined

    When a value is absent or a variable hasn't been given a value in JavaScript, the two separate values null and undefined are frequently used to represent these situations. They differ in their use cases despite sharing some similarities:

  • null :
  • null is a primitive value in JavaScript
  • It is often used to explicitly indicate the absence of a value or to reset a variable or object property to a state with no value
  • Developers purposefully assign the value null, as opposed to undefined, to denote that a variable or property has no meaningful value.
  • undefined :
  • undefined is also a primitive value in JavaScript
  • It is often automatically assigned to variables that have been declared but not initialized with a value.
  • Functions that do not explicitly return a value will also return undefined by default.

    • Question : What do you mean by REST API?

    REST API stands for Representational State Transfer Application Programming Interface. It is a set of architectural principles and constraints for designing and interacting with web services. REST is not a technology or a protocol itself but rather a set of guidelines and best practices for creating web services that are scalable, maintainable, and easy to understand.
